Bandit wargame

0.第一关根据题目提示利用ssh连接到主机 bandit.labs.overthewire.org，在端口2220的用户名是bandit0，密码为bandit0
命令为：ssh bandit0@bandit.labs.overthewire.org -p 2220

0-1.下一关的密码藏在readme文件中，直接使用cat命令查看

1-2.利用0-1得到的密码连接到本关的主机，发现第三关的密码藏在“-”中，“-”在linux中是有特殊意义的，如果直接使用cat查看的话无法读取，所以使用“./-”来表明是作为路径查看的。

2-3.利用1-2得到的密码连接到本关的主机，发现下一级别的密码存储在主目录中此文件名中称为空格的文件中，这道题文件名中有空格，可以用双引号把文件名包裹起来

3-4.利用2-3得到的密码连接到本关的主机,下一级别的密码存储在inhere目录中的隐藏文件中,在linux中，文件名前面有.的就是隐藏文件，可以使用ls -a来显示

4-5.利用3-4得到的密码连接到本关的主机,下一关的密码保存在inhere文件夹下唯一可读的文件中,使用 file 命令可以查看所有文件的文件类型,我个人觉得可以挨个打开一遍也能找得到密码

5-6.利用4-5得到的密码连接到本关的主机,这关是在inhere目录下找到一个可读，大小为1033字节且不可执行的文件，用到 find 命令。参数 -size 可以用来指定文件的大小，在数字后加 ‘c’用字节表示，加’k’用 kb 表示，加’M’用 MB 表示。参数 -type 用来指定文件的类型，’f’表示普通文件，’I’表示链接文件，’d’表示目录。

6-7.利用5-6得到的密码连接到本关的主机,这一关的文件有以下特征：属于用户 bandit7；属于组 bandit6,33字节,find命令, -group bandit6 将会寻找属于组 bandit6 的文件。参数 -user bandit7 将会寻找属于用户 bandit7 的文件。因为该文件在服务器的某处，所以我们要从根目录开始找。执行命令后将会产生许多拒绝访问的报错信息。为了过滤掉这些没用的信息，我们使用 （2>/dev/null）

7-8.利用6-7得到的密码连接到本关的主机,题目说密码在data.txt中并且在单词millionth的后面，可以grep “millionth” data.txt或者cat data.txt|grep millionth

8-9.利用7-8得到的密码连接到本关的主机,下一关的密码保存在文件 data.txt中并且仅有一行只出现一次，uniq -u仅显示出一次的行列，但是使用之前需要用sort命令对文本进行排序，因为uniq命令是通过判断上下两行是否一样来判断的，所以用sort排序一下然后在uniq就能找到唯一出现的那一行了

9-10.利用8-9得到的密码连接到本关的主机,下一关的密码保存在 data.txt 文件中，只有少部分以’=’ 开头的字符串是可读的，用cat命令之后会出现很多乱码，因此需要使用strings命令，获取可打印的字符。使用：strings data.txt

10-11.利用9-10得到的密码连接到本关的主机,下一级别的密码存储在文件data.txt中，该文件包含base64编码的数据，利用base64 -d直接解码

11-12.利用10-11得到的密码连接到本关的主机,下一级别的密码存储在文件data.txt中，其中所有小写（az）和大写（AZ）字母都已旋转了13个位置，就相当去26个字母的前13个位置与后13个位置调换了。那么我们就是用tr命令进行调换，cat data.txt | tr ‘a-zA-Z’ ‘n-za-mN-ZA-M’

12-13.利用11-12得到的密码连接到本关的主机,下一级别的密码存储在文件data.txt中，该文件是已重复压缩的文件。对于此级别，在/ tmp下创建一个目录，在其中可以使用mkdir进行工作可能很有用。例如：mkdir / tmp / myname123。然后使用cp复制数据文件，然后使用mv重命名该数据文件。首先按照提示，在/tmp目录下创建自定义的文件夹，然后我们发现data.txt是一个hex dump文件，里面是十六进制的内容，可以用xxd命令将其转换成二进制文件，然后用file命令看一下这个二进制是什么文件，发现是一个gzip压缩文件，那么利用mv命令把文件重命名，然后用gzip -d命令解压，发现还是一个二进制文件，继续file命令查看，不断重复，最后得到密码。

13-14.利用12-13得到的密码连接到本关的主机,下一级别的密码存储在/etc/bandit_pass/bandit14中，并且只能由bandit14用户读取。对于此级别，您不会获得下一个密码，但是会获得一个可用于登录下一级别的专用SSH密钥。这道题我们使用bandit13用户登陆的，但是题目说需要我们用bandit14用户登陆才能查看密码，并且给了我们ssh的私钥，那么我们就可以利用ssh -i参数指定私钥进行登陆：ssh -i sshkey.private bandit14@localhost，再用cat打开/etc/bandit_pass/bandit14即可。

14-15.上一题已经连接到本题主机，可以通过将当前级别的密码提交到localhost上的端口30000来检索下一级的密码，用telnet连接到本地的30000端口上，然后把这关的密码发送过去。

15-16.利用14-15得到的密码连接到本关的主机,可以通过使用SSL加密将当前级别的密码提交到localhost上的端口30001来检索下一级别的密码，openssl s_client -connect localhost:30001

16-17.利用15-16得到的密码连接到本关的主机,可以通过将当前级别的密码提交到本地主机上31000到32000范围内的端口上的密码来检索下一级的凭据。首先找出这些端口中的哪些服务器正在侦听它们。然后找出其中哪些使用SSL，哪些不使用SSL。只有一台服务器将提供下一个凭据，题目中提示了nmap，直接用nmap扫描端口，nmap -p 31000-32000 localhost，发现两个端口，用openssl s_client -connect localhost:检查两个端口号，发现31790是我们需要的，另外，我们可以写一个脚本，创建一个 /tmp/key 文件夹并将密钥保存在这：cat /etc/bandit_pass/bandit16 | openssl s_client -connect localhost:31790 -quiet > /tmp/key/b16pkey，然后登陆：ssh -i /tmp/key/b16pkey bandit17@localhost，用cat打开：cat /etc/bandit_pass/bandit17

Linux常用命令

开关机命令

1、shutdown –h now：立刻进行关机

2、shutdown –r now：现在重新启动计算机

3、reboot：现在重新启动计算机

4、su -：切换用户；passwd：修改用户密码

5、logout：用户注销

常用快捷命令

1、tab = 补全

2、ctrl + l -：清屏，类似clear命令

3、ctrl + r -：查找历史命令（history）；ctrl+c = 终止

4、ctrl+k = 删除此处至末尾所有内容

5、ctrl+u = 删除此处至开始所有内容

展示目录列表命令ls（list）

1、ls 展示当前目录下的可见文件

2、ls -a 展示当前目录下所有的文件（包括隐藏的文件）

3、ls -l(ll) 展示当前目录下文件的详细信息

4、ll -a 展示当前目录下所有文件的详细信息

5、ll -h 友好的显示当前目录下文件的详细信息（其实就是文件的大小可读性更强了）

切换目录命令cd（change directory）

1、cd test 切换到test目录下

2、cd .. 切换到上一级目录

3、cd / 切换到系统根目录下

4、cd ~ 切换到当前用户的根目录下

5、cd - 切换到上一级所在的目录

目录的创建（mkdir）和删除（rmdir）命令

1、mkdir test 在当前目录下创建一个test目录

2、mkdir -p test/a/b 在test目录下的a目录下创建一个b目录，如果上一级目录不存在，则连它的父目录一起创建

3、rmdir test 删除当前目录下的test目录（注意：该命令只能够删除空目录）

文件打包或解压命令tar

Linux中的打包文件一般是以.tar结尾的，压缩的命令一般是以.gz结尾的,而一般情况下打包和压缩是一起进行的，打包并压缩后的文件的后缀名一般.tar.gz。

tar -zcvf 打包压缩后的文件名 要打包压缩的文件

tar [-xvf] 压缩文件

端口管理

nststat:查看端口开放情况

-a:表示列举所有的连接、服务器监听

-t:列出所有tcp协议的服务

-u:列出所有udp协议的服务

-n:使用端口号来显示

-l:列出所有的监听

-p:列出所有服务的进程id（pid）

常用：netstat -atunlp

进程管理：ps

ps:查当前进程

ps -ef | grep java:查看java的进程

jps:==(ps -ef | grep java) 只有在linux中安装了JDK才能用

网络管理：ping、ifconfig

服务管理命令：

service:

格式：
service s_name start|stop|status|restart

linux系统所有自带服务名称：/etc/init.d/

常用：

关闭防火墙服务
:service iptables stop

重启网络服务：
service network restart

mysql数据库服务的名称：

mysql版本低于5.5 mysqld

mysql版本高于5.5 mysql

more

在 more 这个程序的运行过程中，你有几个按键可以按的：

空白键 (space)：代表向下翻一页

Enter ：代表向下翻『一行』

/字串 ：代表在这个显示的内容当中，向下搜寻『字串』这个关键字

:f ：立刻显示出档名以及目前显示的行数

q ：代表立刻离开 more ，不再显示该文件内容

b 或 [ctrl]-b ：代表往回翻页，不过这动作只对文件有用，对管线无用
